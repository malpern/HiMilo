Network & Remote Setup

VoxClaw includes a local HTTP listener that receives text from your coding agent. This is how the Claude Code and Codex plugins communicate with VoxClaw.

Local Use (Default)

When your agent runs on the same Mac as VoxClaw, everything works over localhost:4140. The network listener is enabled automatically when you install a plugin.

Remote Agent

If your agent runs on a different machine (e.g., a Mac Mini or server), VoxClaw can listen for text over the local network:

  1. Enable the network listener in Settings
  2. Note your Mac's local IP address (shown in Settings)
  3. Configure your agent to send text to http://<your-ip>:4140/read

HTTP API

POST /read

Send text to be spoken aloud.

curl -X POST http://localhost:4140/read \
  -H 'Content-Type: application/json' \
  -d '{"text": "Hello from your agent"}'

POST /ack

Acknowledge the current reading (stop it).

curl -X POST http://localhost:4140/ack \
  -H 'Content-Type: application/json' \
  -d '{"project_id": "/path/to/project"}'

GET /status

Check if VoxClaw is running and its current state.

curl http://localhost:4140/status

Custom Port

Change the listener port in Settings or with the VOXCLAW_PORT environment variable. The default is 4140.

Firewall

If your agent is on another machine, make sure your Mac's firewall allows incoming connections on the VoxClaw port. macOS may prompt you to allow this when you first enable the listener.
