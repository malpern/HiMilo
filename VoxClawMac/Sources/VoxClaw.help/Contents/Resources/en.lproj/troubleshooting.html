Troubleshooting

VoxClaw isn't speaking

  • Check that VoxClaw is running (look for its icon in the menu bar)
  • Verify the listener is active: curl http://localhost:4140/status
  • Make sure your volume isn't muted
  • If using OpenAI/ElevenLabs, verify your API key is valid in Settings

Agent responses aren't triggering speech

  • Restart your Claude Code or Codex session after installing the VoxClaw plugin
  • Check that hooks are installed: look for voxclaw entries in ~/.claude/settings.json or ~/.codex/config.toml
  • Enable the network listener in VoxClaw Settings

Speech cuts off mid-sentence

  • This usually means the UserPromptSubmit hook fired (you typed a new message)
  • If it happens without typing, check for other applications sending to VoxClaw's port

OpenAI voices aren't working

  • Verify your API key at platform.openai.com
  • Check that you have available credits in your OpenAI account
  • VoxClaw will fall back to Apple's built-in voice if the OpenAI request fails

Peer device not appearing

  • Both devices must be on the same Wi-Fi network
  • VoxClaw must be running on both devices
  • Check your firewall settings — Bonjour discovery uses mDNS
  • Wait a few seconds for Bonjour to resolve the peer address

Overlay not showing

  • Check that "Audio only" is not enabled in Settings
  • The overlay may be positioned off-screen — try toggling "Remember overlay position" off and back on

View Logs

For detailed diagnostics, view VoxClaw's logs in Terminal:

log stream --predicate 'subsystem == "com.malpern.voxclaw"' --level debug
