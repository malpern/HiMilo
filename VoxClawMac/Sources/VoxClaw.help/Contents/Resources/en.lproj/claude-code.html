VoxClaw Help › Claude Code Setup

Connect to Claude Code

The VoxClaw plugin for Claude Code makes every response spoken aloud automatically. When you send a message, VoxClaw stops reading the previous response so it never talks over you.

Install via Plugin Marketplace

Run these two commands in your terminal:

claude plugin marketplace add malpern/VoxClaw
claude plugin install voxclaw

That's it. New Claude Code sessions will pick up the plugin automatically.

Manual Install

If you prefer to install hooks directly:

git clone https://github.com/malpern/VoxClaw.git
bash VoxClaw/plugins/voxclaw/setup-claude-code.sh

This copies hook scripts to ~/.claude/hooks/ and adds entries to ~/.claude/settings.json.

What the Plugin Does

Stop Hook

When Claude Code finishes responding, the plugin extracts the response text, strips markdown formatting, and sends it to VoxClaw's local listener to be spoken aloud.

UserPromptSubmit Hook

When you send a new message, the plugin tells VoxClaw to stop reading the previous response. This means VoxClaw never talks over your next interaction.

Multi-Agent Voices

If you have multiple Claude Code sessions running in different project directories, VoxClaw assigns each project a distinct voice automatically. Responses are queued — one project speaks at a time, with smooth transitions between them.

Verify It Works

Start a new Claude Code session and ask it a question. You should hear the response spoken aloud and see the teleprompter overlay appear.

Running sessions need to be restarted to load newly installed hooks.
